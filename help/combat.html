Combat

Attacks

Setting up attack skills/spells are done with the /attack_method and the related /attack_skill, /attack_spell commands. First decide if your main attack method is use or cast. Next set your attack skill/spell using the appropriate commands. There are also /start_attack_skill and /start_attack_spell commands. For example, if you were a Monk you might set the following:

/attack_method use
/attack_skill kungfu
/start_attack_skill ki

You can then use the /k (which uses the skill/spell), /kk (same as /k but also uses the kill command against the target), /ka (same as /kk but uses kill all command) and /ks (similar to /k except uses the start attack skill/spell). You may also be interested in checking out /do_kill.

Heals

Setting up heal skills/spells are done with the /heal_method and the related /heal_skill, /heal_spell commands. First decide if your main heal method is use or cast. Next set your heal skill/spell using the appropriate commands. The heal macros are set up to make use of chains. When you heal a new target the _healing alias will be updated. For example, if you were a Cleric you might set the following:

/heal_method cast
/heal_spell half heal
chain half heal:half heal:_healing

You can then use the /h (which uses the skill/spell). You may also be interested in checking out /do_heal.

Looting and Corpses

The overall corpse setting is /on_kill. All of the various corpse/loot trigs will check the value of on_kill before checking their individual settings.

The /on_kill_corpse command will take any of the following arguments: leave, eat, tin, dig, carriage, get and stuff. Some of the commands require items in order to work properly, such as a tinning kit or shovel.

The /on_kill_loot command is a boolean which will enable/disable looting of corpses. It uses the mud's autoloot setting as well as using loot as a backup.

The /on_kill_give_to command defines a target which should receive your loot. This could be another player, a pet, a mule, etc.

The /on_looted command works of a different message. It will attempt to activate the /loot macro after somebody else loots. This helps to avoid possible issues with looting before somebody who has dibs on the body.
